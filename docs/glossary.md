# Glossary

| Term | Definition |
|---|---|
| **ADR** | Architecture Decision Record. We use [MADR](https://adr.github.io/madr/). See [docs/adr/](adr/). |
| **Agent task contract** | A YAML file under `tasks/` instantiating the schema (I, O, S, R, T, Φ, Ψ). The unit of work an AI agent can claim. |
| **Allowlist** | `policies/agent-allowlist.yml` — paths an agent may freely edit. |
| **AGENT_CARD** | Per-agent capability documentation per Mitchell et al. (model cards) and EU AI Act Annex IV. [AGENT_CARD.md](../AGENT_CARD.md). |
| **AGENTS.md** | Universal AI-agent contract per the [agents.md](https://agents.md) spec. [AGENTS.md](../AGENTS.md). |
| **Audit chain** | Hash-chained, HMAC-signed log of every state-changing action. [docs/audit/audit-log-spec.md](audit/audit-log-spec.md). |
| **Catalog** | A signed manifest listing modules, published by a `publisher` with a declared `trust_level`. See [module-system.md](architecture/module-system.md). |
| **CODEOWNERS** | GitHub feature: routes review requests by changed path. We use multi-team lines to enforce two-team approval on sensitive paths. |
| **Cosign** | Sigstore tool for signing/verifying artifacts with short-lived OIDC-bound certs. |
| **DSSE envelope** | Dead-Simple Signing Envelope — the format used for in-toto attestations. |
| **gitsign** | Sigstore tool that signs Git commits/tags with keyless OIDC instead of long-lived GPG keys. |
| **HMAC** | Keyed-hash message authentication. We use HMAC-SHA256 with OIDC-bound KMS keys to sign each audit row. |
| **in-toto** | Supply-chain provenance framework. Provenance is emitted as DSSE envelopes with the `https://slsa.dev/provenance/v1` predicate. |
| **Kolibri** | Open-source offline learning platform from Learning Equality, shipped on the OS for offline content delivery. |
| **KMS** | Cloud Key Management Service (AWS KMS, GCP KMS, or equivalent). Holds our HMAC key; issues short-lived access via OIDC. |
| **Live-build** | Debian's declarative tool for building live ISO images. See [Debian Live Manual](https://live-team.pages.debian.net/live-manual/html/live-manual.en.html). |
| **MADR** | Markdown Any Decision Records — the ADR template format we use. |
| **Mergify** | Auto-merge bot. Enforces queue-only merging once required approvals + checks are green. |
| **Module manifest** | YAML file describing a learning module: id, version, language, age_band, content, license, checksum, signature. |
| **OIDC** | OpenID Connect. Used by GitHub Actions and developer OAuth to assert workload identity to Sigstore/KMS without long-lived secrets. |
| **Planner / Builder / Reviewer subagents** | Specialized Claude Code subagents under `.claude/agents/` with constrained tool access. |
| **Protected path** | A path in `policies/protected-paths.yml` `deny:`. Agents cannot edit without `allowlist-override` + two-team approval. |
| **Rekor** | Sigstore's transparency log. All gitsign/cosign signatures are recorded here, append-only, publicly verifiable. |
| **Renovate** | Dependency-update bot; we use it for Python/Node/container deps (Dependabot handles GH Actions). |
| **Ruleset** | GitHub's newer branch-protection mechanism, superseding classic branch protection rules. Our exports live in `.github/rulesets/`. |
| **SBOM** | Software Bill of Materials. We use CycloneDX format generated by Syft. |
| **Sensitive change** | A change touching paths listed in `policies/sensitive-change-labels.yml`. Auto-labeled; requires two-team approval. |
| **Skill** | Reusable Claude Code workflow stored under `.claude/skills/<name>/SKILL.md`. Invocable on demand. |
| **SLSA** | [Supply-chain Levels for Software Artifacts](https://slsa.dev). We target L2+ for releases. |
| **SMO-NNNN** | Task ID format. `SMO` = Shikshan Mantra OS; `NNNN` = 4-digit sequence number. |
| **Snapshot.debian.org** | Time-stamped archive of Debian packages. Pinning to a snapshot is what makes our builds reproducible. |
| **Subagent** | A specialized Claude Code agent invoked by the main agent. Has its own tool allowlist and prompt. |
| **Sync endpoint** | Optional school/community server URL for pushing progress data. Off by default; configured in admin policy. |
| **Task queue** | File-system-backed queue under `tasks/{open,in-progress,completed,blocked}/`. An agent picks a task by moving its YAML between directories. |
| **Two-person rule** | Sensitive-path edits require approvals from two distinct GitHub accounts AND two distinct CODEOWNERS teams. Enforced by Rulesets + CODEOWNERS. |
| **WebXR** | Web standard for VR/AR experiences. Our AR labs use it with a 3D-desktop fallback when AR hardware/browser support is unavailable. |
| **Workflow `name:`** | The string that becomes a GitHub status-check context. Required to match the corresponding entry in `.github/rulesets/main-branch.json` — this is the binding contract. |
| **Ψ recovery** | The recovery clause of a task contract: what the agent does when a budget is exceeded, a protected path is hit, or tests fail repeatedly. |
| **Φ preconditions** | What must be true before an agent claims a task (label present, ADR exists, audit chain clean, scope is allowlist-compatible). |
