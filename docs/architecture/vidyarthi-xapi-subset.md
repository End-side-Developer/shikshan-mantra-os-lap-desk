## Overview

Every Vidyarthi engine returns an array of xAPI statements as part of
its `grade` RPC response. The launcher appends them to
`~/.local/share/shikshan/vidyarthi/learner.db` (SQLite, append-only)
and anchors the session into the audit chain at session close.

This document defines the **mandatory fields** and the **allowed
verb/activity-type URIs** for Vidyarthi v1. Any field not listed here
is optional and may be added by individual engines in `context.extensions`.

---

## Mandatory fields

```json
{
  "id": "<uuid-v4>",
  "actor": {
    "objectType": "Agent",
    "account": {
      "homePage": "urn:shikshan:device",
      "name": "<device-scoped-uuid>"
    }
  },
  "verb": {
    "id": "<verb-uri>",
    "display": { "en-US": "<verb-label>" }
  },
  "object": {
    "objectType": "Activity",
    "id": "urn:shikshan:module:<module-id>:exercise:<exercise-id>",
    "definition": {
      "type": "http://adlnet.gov/expapi/activities/assessment",
      "name": { "en-US": "<exercise title>" }
    }
  },
  "result": {
    "score": {
      "scaled": 0.0,
      "raw": 0
    },
    "success": false,
    "response": "<learner-submitted-text, max 4 KB>"
  },
  "context": {
    "platform": "Shikshan Mantra OS",
    "language": "en",
    "extensions": {
      "urn:shikshan:sub_engine": "sql",
      "urn:shikshan:engine_version": "1.0.0"
    }
  },
  "timestamp": "2026-05-28T00:00:00.000Z"
}
```

### Field semantics

| Field | Rule |
|---|---|
| `id` | UUID v4 generated by the engine for this statement. Globally unique. |
| `actor.account.homePage` | Always the literal string `"urn:shikshan:device"`. |
| `actor.account.name` | A UUID v4 generated once at Vidyarthi first-launch, stored in `~/.config/shikshan/vidyarthi/device.uuid`. Never rotated unless the user explicitly resets progress. **No email, no name, no institutional ID** unless `auth.yml` `modes` includes `institution` (future). |
| `verb.id` | One of the URIs from the table below. |
| `object.id` | Stable URN using the module `id` and exercise `id` from the manifest. Stable across exercise version bumps unless the exercise is replaced. |
| `result.score.scaled` | Float in [0.0, 1.0]. `raw / 100`. |
| `result.score.raw` | Integer in [0, 100]. The `grade` RPC `score` field. |
| `result.success` | Boolean. True iff `score >= rubric.pass_threshold`. |
| `result.response` | The learner's submission text, truncated to 4 096 bytes. |
| `context.language` | Active UI locale at time of submission (`"en"` or `"hi"`). |
| `context.extensions["urn:shikshan:sub_engine"]` | The active sub-engine name (e.g. `"sql"`). |
| `context.extensions["urn:shikshan:engine_version"]` | The engine binary's SemVer string. |
| `timestamp` | ISO 8601 UTC. Set by the engine process at grade time. |

## Allowed verbs

| Verb URI | Display (en) | When to emit |
|---|---|---|
| `http://adlnet.gov/expapi/verbs/attempted` | attempted | Every time the learner presses "Run" (run RPC). |
| `http://adlnet.gov/expapi/verbs/completed` | completed | When `success = true` for the first time on this exercise. |
| `http://adlnet.gov/expapi/verbs/scored` | scored | Always included in the `grade` RPC response alongside `attempted` or `completed`. |

A single `grade` call typically emits two statements:
one `scored` (always) + one `completed` (if passing for the first time).

## Storage schema

```sql
CREATE TABLE IF NOT EXISTS xapi_statements (
    id         TEXT PRIMARY KEY,
    stored_at  TEXT NOT NULL,
    statement  TEXT NOT NULL
);
```

- `id` â€” the xAPI statement UUID.
- `stored_at` â€” ISO 8601 UTC wall-clock time when the launcher wrote the row.
- `statement` â€” full JSON blob.
- Append-only. The launcher MUST NOT UPDATE or DELETE rows.

## Audit-chain anchoring

At session close the launcher calls `scripts/audit/append-entry.py`:

```yaml
action:   "xapi-session-close"
subject:  "<module-id>/<exercise-id>"
detail:   "sha256:<hex-of-sha256(sorted(statement_uuids_in_session))>"
```

This anchors the session into the hash-chained audit log without
embedding statement payloads in `docs/audit/audit.db`.

---

## Future extensions

Extensions not in this subset are stored as-is in `context.extensions`
and MUST use the `urn:shikshan:` namespace prefix. Third-party engines
MUST use their own namespace prefix (e.g. `urn:mypublisher:`).

LRS export (batch-submit to an external xAPI LRS) is a post-MVP feature
(post-3-engines milestone).
