# Security Policy

## Supported versions

Pre-release. Once v1.0.0 ships, the most recent minor release and the previous minor release receive security fixes.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for vulnerabilities. Instead:

1. Email `security@shikshan-mantra.example` (PGP key fingerprint published before first release).
2. Or use GitHub's private vulnerability reporting on this repository.

Include: affected component, reproduction steps, observed vs expected behavior, and your suggested severity. We acknowledge within 72 hours and provide a remediation timeline within 7 days.

## Scope

In scope:
- The built ISO and any package built from this repository
- All scripts under `scripts/`, build hooks under `config/hooks/`, and live-build configuration
- Module manifest, catalog manifest, progress record, and admin policy schemas
- The audit log (`docs/audit/audit.db`) and its append/verify scripts
- CI workflows in `.github/workflows/` and rulesets in `.github/rulesets/`
- All AI-agent governance surfaces: `AGENTS.md`, `CLAUDE.md`, `AGENT_CARD.md`, `policies/`

Out of scope:
- Vulnerabilities in upstream Debian packages — please report to Debian's security team. We will pull fixes when they reach the configured repositories.
- Bugs in third-party learning content provided by community catalogs — report to the catalog publisher first.

## Signing & provenance

- Every commit is signed via [Sigstore gitsign](https://docs.sigstore.dev) (keyless OIDC).
- Every release artifact (ISO, SBOM, attestation) is signed via [Cosign](https://docs.sigstore.dev/cosign/overview/) keyless.
- SLSA L2+ in-toto provenance is generated by `release-slsa.yml` and attached to every GitHub Release.
- Public verification recipe: [docs/security/slsa-provenance.md](docs/security/slsa-provenance.md).

## Key custody

- No long-lived signing keys are stored in the repository or in CI secrets.
- Sigstore short-lived certificates are issued per-run via GitHub OIDC.
- Debian archive keys under `config/archives/*.key.*` are stored as binary files with `signed-commit-required` enforcement; rotation per [docs/runbooks/rotate-signing-key.md](docs/runbooks/rotate-signing-key.md).

## Threat model

See [docs/architecture/threat-model.md](docs/architecture/threat-model.md). It covers:
- STRIDE for each component (live-build pipeline, module catalog, progress store, sync, AI assistant)
- OWASP Top 10 for Agentic Applications (2025) mappings for the AI-agent development surface
- Supply-chain risks (dependencies, base image, kernel) and mitigations (SBOM, Grype, pinning to snapshot.debian.org)

## Coordinated disclosure

We follow a 90-day coordinated disclosure window unless the vulnerability is already public or being exploited. Credit is published in the release notes unless the reporter requests anonymity.
